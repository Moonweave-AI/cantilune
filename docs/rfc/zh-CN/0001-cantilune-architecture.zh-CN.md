## 1. 摘要

`cantilune` 是一个开源的、与模型能力解耦的 agent 编排框架，其核心是一个**统一的形式结构**，将四种形式体系作为四个侧面纳入其中：**DAG**（呈现 / 数据流清晰性）、**Petri 网**（分布式系统 / 并发本质）、**π 演算**（通信本质）、以及**范畴论态射**（复合/映射本质）。本框架针对现有编排器中四种被明确记录的失败模式（Cursor、Codex、Claude Code、OpenClaw 系列），并使其优越性在四个可测量的维度上可证伪。

## 2. 动机

### 2.1 问题

现有 agent 编排系统表现出四种被命名的失败模式，每一种都与对数学/逻辑结构的忽视相关：

| 系统 | 失败模式 | 结构性根因（假设） |
|---|---|---|
| Cursor | 固化的 plan→subagent→loop $\Rightarrow$ 表达力有限 | 固化的流程形态；未解耦的数据依赖图 |
| Codex | A2A + leader/presenter $\Rightarrow$ 流程不可控、周期不可预知 | 缺乏形式化的并发/终止语义；leader 是模型驱动的瓶颈 |
| Claude Code | 厚控制面约束 agent $\Rightarrow$ 效果不尽人意 | 结构化调度与模型决策之间缺乏清晰边界 |
| OpenClaw 系列 | 冗杂代码 $\Rightarrow$ 工程困顿 | 缺乏精简的形式化核心；临时性堆积 |

### 2.2 受益人 / 为何现在 / 不做的代价

- **受益人：** 需要可移植、厂商中立的编排核心的 agent 系统构建者；触及固定循环工具能力上限的团队；随着 agent 分支日益庞大而受益的更广泛生态。
- **为何现在：** 生态正在快速分支化；在封闭的事实标准固化之前确立一个有原则的参考实现的窗口正在关闭。
- **不做的代价：** 厂商锁定持续；每个团队各自重新实现临时编排；上述结构性失败被复制而非被解决。

### 2.3 后果

如果没有一个形式化奠基的核心，`cantilune` 将复现它所批判的失败（尤其是 OpenClaw 的臃肿）。**结构本身就是产品。** 这个结构*究竟是什么*——以及它为何不是对图形式体系的随意选择、而是编排本身的数学本质——是 §6.1 的主题。

## 3. 目标

1. 定义一个**统一的形式结构**，其中 DAG、Petri、π 演算、态射是投影/侧面，而非彼此替代。
2. **将编排与模型能力解耦**：调度/路由由结构 + 策略驱动；模型被限制在节点内部工作。
3. 使**可观测性成为结构性属性**：每次执行都是图的一个确定性投影；trace/重放是一等公民。
4. 相对于被命名的基线，确立**四条可证伪的优越性主张**，并配有评测计划以逐一测量。
5. 保持**工程精简**：一个最小可行核心，分阶段扩展，以规避 OpenClaw 失败模式。

## 4. 非目标

- 不是围绕单一厂商 agent 循环的包装器。
- 不是对冗杂代码库的重新实现；没有分阶段正当理由的"完整框架"范围蔓延。
- 不闭源、不固化流程。
- 不是模型训练或模型托管系统。
- 不是面向终端用户的产品 UI（参考实现可含最小化的 CLI/工具；UX 不在本 RFC 范围内）。

## 5. 背景

### 5.1 统一结构论点（来自 triage 决策）

四种形式体系各占一个侧面；`cantilune` 是这些侧面**所归属的那个更高阶结构**：

| 形式体系 | 侧面 | 所提供之物 |
|---|---|---|
| DAG | 呈现 / 数据流清晰性 | 可读的拓扑、显式数据依赖、"什么依赖于什么" |
| Petri 网 | 分布式系统 / 并发本质 | 并发、资源边界、终止/活性 |
| π 演算 | 通信本质 | 形式化的 A2A 消息、动态拓扑、通道移动性 |
| 态射（范畴论） | 复合 / 映射本质 | agent/操作如何复合的极简精确定义 |

每一个都是一面**透镜**；统一结构是这些透镜**所透射之物**。框架必须具体地定义这个更高阶结构（而非仅断言其存在）——这是 ADR-0001 的核心任务。

关于此论点有两点易被误读，值得预先说明，以便 §6.1 落得准确。其一，"每种形式体系各占一面"**不**意味着"我们把四个模块拼起来"。本项目旨在避免的反复出现的失败模式，正是把 DAG、Petri、π、态射当作四个独立代码模块、由适配器粘合——那会复现 OpenClaw 的臃肿，更糟的是使"四视角是否一致"这个问题没有形式答案。论点恰恰相反：只有*一个*对象，四种形式体系是它的四种*读法*。§6.1 将此讲精确——编排*即*一个对称幺单范畴，四个投影是从它出发的保结构函子，而"它们一致"成为一条定理（RFC-0002 的主题），而非一种希望。

其二，上表是*地图*，非*疆域*。它说明哪种形式体系照亮哪个侧面，使读者知道形式故事的每一块将来自何处；真正的定义、四读法一致的证明、以及精确对象，在 `docs/spec/formal-semantics.md`。偏好先直觉后形式的读者，可从**通俗读本**（`docs/primer/for-the-curious.zh-CN.md`）起步，它以不带符号的方式解释同样的想法，并链接回此处。

### 5.2 相关工作（定位 —— 评审期间扩展）

**未经验证** —— 以下定位是关于各系统**已声明形态**的事实，不构成背书；评审期间必须核实。

| 系统 | 关系 | Cantilune 的楔子（差异点） |
|---|---|---|
| **LangGraph** | 最近的先例（基于图的 agent 图） | LangGraph *仅是* DAG 侧面；cantilune 统一了 DAG + Petri（资源/并发）+ π 演算（通信）+ 态射（复合）。组合楔子：(a) 类型化数据契约边 + 资源库所，(b) 模型解耦的结构化路由，(c) 可观测性即结构。 |
| **AutoGen / CrewAI** | 多 agent A2A | π 演算侧面提供 AutoGen/CrewAI 所缺乏的形式化通信语义；结构化路由 vs leader 瓶颈针对 Codex 失败模式。 |
| **A2A 协议** | 通信标准 | cantilune 的通信侧面应**与 A2A 兼容**作为传输层，其上叠加 π 演算作为形式语义。 |
| **MCP** | 工具调用标准 | cantilune 节点应可调用 MCP 工具；MCP 是节点内部能力，而非编排原语。 |
| **Temporal / Airflow / Dagster** | DAG 工作流（非 agent） | cantilune 共享 DAG 拓扑但增加 agent 专用侧面（通信、模型解耦路由、agent 可观测性）。这些系统是调度器设计的数据面参考。 |

> **进入 FCP 前必须填补的关键缺口：** 具体验证 LangGraph 差异点。"LangGraph 缺少 X" 必须对照 LangGraph 当前实际能力核查，而非假设。

## 6. 提案

### 6.1 为什么编排是一个数学对象（统一结构）

§2 列出四种失败模式，但它们共享一个比任何单一缺陷都更深的根：**现有编排器没有统一的形式语义层**。编排的*结构*——什么依赖什么、什么并行、谁与谁通信——和它的*执行*——如何推进、是否会卡死、能否重放——是用两种互不相干的语言描述的：结构是一张图或一份配置，执行是某人那周写的临时调度器代码。后果是，每一项新能力都要重造一套执行语义，而每一个保证（"不死锁""可重放""可审计"）都要从头论证，且论证与代码会随时间漂移。这正是这些系统最终或固化（Cursor）、或失控（Codex）、或被厚控制面撑开（Claude Code）、或被自身语义债务压垮（OpenClaw）的原因。缺的不是更好的图模型，而是**一个同时就是结构和执行的对象**。

什么能统一结构与执行？答案不是"选一个更强的图形式体系"，而是一个更基本的观察：**编排本身就是一个对称幺单范畴（SMC）**。编排最基本的两件事——"并行做两件事"和"串行做两件事"——恰好就是张量积 $\otimes$ 与复合 $\circ$；而它们已然满足的法则（并行工作可重排、串行可结合、空操作是单位）恰是 SMC 公理。这不是"我们为编排选了 SMC 这个工具"，而是"**编排*即* SMC**"——SMC 是编排的数学本质，正如关系代数是数据库的数学本质。这一等同是整个项目的支点：一旦编排是 SMC，它的执行就应是*该 SMC 上的动力学*，而非在它之外另写一套调度器代码去谈论它。

仅 SMC 只给静态结构——*哪些*复合合法。要说它*怎么*跑，需加一层动力学。我们选**弦图重写**：一次执行步即"按一条规则把一个小子图替换为另一个"。为何是重写而非更朴素的状态机？因为只有重写，能把"四个投影看到的都是*同一次*执行步"从"在四套状态机之间手工对齐的妥协"变成"一条定理"——而那手工对齐正是 ADR-0001 所警告的"四个互不相连模型"失败模式。于是统一对象定为 $\text{CantiluneGraph} = (C, R)$：SMC $C$ 是结构，重写 $R$ 是执行。一个对象，两种角色，没有第二套语言。

那么 DAG、Petri、π、态射又是什么？不是四个模块——**是看同一 $(C, R)$ 的四个视角**，正如一个数据库可经关系代数、经事务、经存储、经并发四个角度去看。DAG 视角读出数据依赖（什么产出什么）；Petri 视角读出并发与资源（什么可以同跑、什么正占着资源）；π 视角读出通信（谁在与谁说话）；态射视角读出复合与重构（小件如何成大件、如何换一件）。同一 $(C, R)$，四种读法：

```
                    CantiluneGraph  =  ( C, R )
                   /        |        |        \
             DAG 投影   Petri 投影  π 演算投影  态射投影
            （数据流）  （并发）     （通信）    （复合）
```

这正是"四理论是四个观察维度，而非四个模块"（§5.1）的精确含义——它不是口号，而是 $(C, R)$ 的直接推论：每个投影是一个 SMC-函子 $P_i : (C, R) \to \text{Target}_i$，对同一对象做一次保持结构的读法，映入一个目标形式体系。

这一等同给 cantilune *运行时*带来什么？它把"调度器怎么决定"从"模型即兴发挥"变成"结构说了算"：并行调度两节点即应用 $\otimes$；施加数据依赖即应用 $\circ$；资源被占用即某 Petri 库所的 token 耗尽、故变迁不触发；一 agent 向另一 agent 通信即 π 通道上一次交互；一次执行步即一次重写。最重要的是，**可观测性成为结构的副产品**：既然执行是一条重写序列，trace *就是*这条序列，重放 *就是*重跑它，而"四视角看到同一运行"正是重写函子一致性所保证的——这是 §6.3 与 RFC-0002 的主题。控制面边界（§6.2）即此结构驱动调度与模型节点内部工作的接合处。

此处须如实陈述。"编排 $=$ $(C, R)$"这一等同在四条线中的**三条——DAG、Petri、态射——按构造成立**，其投影是标准 SMC-函子（Petri 经 Meseguer–Montanari，配合 spec §10.8/§8.6 的 pre-net 解决）。π 这条线不同：因为我们选了*握手后自由对话*（half-π (II)），其投影需到预层语义的桥接定理，属**待证**，非按构造。故支点在三线上稳固，第四线须*挣得*——这正是 ADR-0001"生死线"与 RFC-0002 的由来。形式定义与证明义务在 `docs/spec/formal-semantics.md`；本 RFC 余下部分以该结构所提供的*能力*及其如何满足 §8 可证伪主张来表述。

### 6.2 模型止于何处、结构始于何处（解耦控制面）

Claude Code 的失败模式不是"控制面太厚"这种可测意义下的厚，而是**没人能说清模型的判断止于何处、结构的权威始于何处**。当每个路由决策、每次重试、每个"该不该 fork 一个子 agent"都由一个只有模型自己读的 prompt 决定时，系统没有可检视的边界——而"更薄"修不了这个，*划线*才能。cantilune 的承诺是一条清晰、能用一句话说尽的线：

> **模型在节点内提议；结构在节点间裁决。**

具体含义：LLM 被限制在*节点内部做什么*——读入、产出、决定调用哪个 MCP 工具来完成工作。节点*之间*的一切——哪些节点并行（$\otimes$）、哪些须等哪些（$\circ$）、资源是否可用（一个有 token 的 Petri 库所）、一个 agent 如何到达另一个（一条 π 通道）——由 $(C, R)$ 加一份声明式策略决定，不由模型。当模型想要超出单节点的自主权——生成子图、开新通道——它以*提议*一次结构变更的方式进行，由策略/结构接受或拒绝；它从不静默重写拓扑。故这条边界不是启发式，而是该对象的属性：

| 决策 | 由谁决定 | 机制 |
|---|---|---|
| 拓扑、数据依赖、并发上界、资源 | **结构** | $(C, R)$（类型化边、库所、通道） |
| 结构未充分指定处的路由/排序 | **策略**（确定性、可检视） | $(C, R)$ 之上的策略规则 |
| 节点内部做什么（实际工作） | **模型** | 单节点内的 LLM/agent |
| 分叉 / 接纳新子图 | **结构 + 策略**，模型*提议* | 模型提议一次重写；策略/结构裁决 |

这正是"控制面精简度"（§8 的 C3 主张）真正所测：不是代码行数，而是*有多少节点间决策是模型驱动的*。cantilune 的回答是"结构允许的范围内尽量少，而结构说清是哪些"。ADR-0002 将形式化此线；本 RFC 只承诺线划在哪里。

### 6.3 为什么可观测性不是功能而是推论

多数系统把可观测性当作事后补加：插桩代码、发射 span、祈祷日志够用。cantilune 不补加可观测性——它*本就有*，因为 $(C, R)$ 本身如此。一旦执行是弦图上的重写序列，别的系统当作独立产品去做的三件事便免费得来：

- **可追踪**是自动的，因为每个节点、边、库所、通道在图中都有稳定标识，而 trace *就是*重写序列——无需额外发射。
- **可重放**是自动的，因为给定 $(C, R)$、策略、以及每个节点记录的模型 I/O，重跑重写序列即可确定性重现该运行。没有会与真实执行器漂移的独立"重放引擎"。
- **可审计**是自动的，因为结构不变量——终止、资源边界、数据契约一致性——是*图本身的性质*，执行前静态可查、执行中运行时可查，而非事后去论证的主张。

这事之所以重要，不在于省工程量（虽然也省），而在于：别的系统里遥测与真相*分叉*——trace 说一套、代码做一套，没人说得清谁对。这里 trace *就是*执行，因为两者都是同一条重写序列。这是 §6.1 等同的直接回报：可观测性即结构不是设计选择，而是"执行 $=$ 重写"所买来的东西。它也是 C1/C2/C3 证据的基础——§8 的可证伪主张正是*从*这些 trace 上测得的，故它们成为可测试的，而非修辞。

### 6.4 运行时暴露什么（接口）

下列四个接口是形式对象与外部世界的接合处。这里以*能力*列出；精确契约延后至后续 ADR，但每个都对应 $(C, R)$ 的一个侧面：

- **图定义 API** —— 声明一个 `CantiluneGraph`：节点、类型化契约边、库所/变迁、通道、态射。这是用户书写静态结构 $C$ 的方式。
- **策略 DSL** —— 声明叠加在图之上的并发/资源/路由策略。这是以确定性而非模型填充节点*之间*未指定空间（§6.2）的方式。
- **节点契约** —— 节点（模型驱动或确定性）须满足的接口；MCP 工具*从节点内*调用，绝非作为编排原语。这是"模型在节点内提议"的形式化。
- **执行器 API** —— 在策略下运行 $(C, R)$ 并发射 trace。这是重写驱动的执行；它发射的 trace 正是 §6.3 所说免费得来的那些。
- **通信传输** —— A2A 兼容的线路协议，其上叠加 π 演算（half-π (II)）作为形式语义。这是 π 投影成为真实网络消息之处。

## 7. 备选方案

| 选项 | 优点 | 缺点 | 裁决 |
|---|---|---|---|
| **不做** | 无工作量 | 失败持续；锁定；生态停滞 | 拒绝 |
| **最小化：仅 DAG** | 最易交付/评测；可观测性清晰 | 无形式化并发/终止（弱于 Codex）；无通信本质（弱于 A2A）；复现 LangGraph 式"仅 DAG" | 拒绝（相对所列基线过弱） |
| **单一形式体系（Petri / π / 态射）** | 聚焦 | 失去其他侧面；每个侧面映射到不同失败模式 | 拒绝（DRI 明确要求四者兼具） |
| **统一结构（DAG+Petri+π+态射）** | 覆盖全部四种失败模式；相对所有基线可辩护；符合 triage 决策 | 规格最复杂；范围/臃肿风险；需要函子一致性证明 | **接受** —— 配合分阶段计划（§10）控制臃肿 |

## 8. 可证伪主张与评测计划（约束"更强大"之断言）

治理禁止无评测的"更强大"主张。每条主张映射到一个基线失败模式与一个可测量标准。**以下所有标准在评测平台建立之前均属未经验证。**

| # | 主张 | 基线 | 指标 / 成功标准（在评测 ADR 中精确化） |
|---|---|---|---|
| C1 | 表达力 | Cursor | 存在 CantiluneGraph 可表示而 Cursor 固化形态不扭曲即无法表示的工作流；计数 + 分类 |
| C2 | 可控性/可预知性 | Codex | 在基准套件上的有界步数/时间（Petri 终止）；固定策略下的确定性推进（依 `docs/spec/formal-semantics.md` §4.3，v0.1 仅提供**步数有界**；时间界不在范围内） |
| C3 | 控制面精简度 | Claude Code | 结构化决策代码与模型决策代码之比；人类评分的"约束开销" |
| C4 | 工程精简度 | OpenClaw | 功能面 / 核心代码复杂度之比；核心圈复杂度 vs 可比功能集 |

这四条主张**是 `cantilune` 被视为成功所必须跨越的门槛**。任一不达都是终止项目的正当理由（依据 idea-triage 的退出纪律）。

## 9. 安全 / 隐私 / 知识产权

- **威胁建模：** 在任何运行时/通信/网络实现之前必需（关卡，当前尚未适用）。
- **不可信输入：** 图、节点载荷、MCP 工具输出、A2A 消息是不可信数据；执行器必须在节点内部强制能力边界（每节点最小权限）。
- **知识产权：** 框架开源（许可证待定）；无溯源审查不引入第三方代码。
- **机密：** 图/trace 中不含机密；trace 不得泄漏可能含用户数据的节点内部模型提示（在可观测性/数据处理 ADR 中处理）。

## 10. AI/Agent 考量

- agent 行为由**结构 + 策略**约束，而非厚控制面（与 Claude Code 直接对比）。
- 高影响 agent 行为（网络、文件写入、工具使用）要求**按节点权限边界**，并整合治理 E-Stop / 安全状态原则（依据 Moonweave 基线）。
- 模型自主性限于节点内部提议；结构裁决。

## 11. 可观测性

见 §6.3。trace 是一等公民；重放是结构性属性。评测（§8）消费 trace。trace schema 是规范性交付物（待定 ADR）。

## 12. 测试 / 评测

- **评测平台**是一等交付物，而非事后补充 —— 它使可证伪主张可被测试。
- 核心不变量（终止、资源边界、数据契约一致性）获得**静态检查**（图级）与**运行时检查**（执行器级）。
- 不捏造结果：任何未测量的主张标记为**未经验证**。

## 13. 推出 / 回滚

- **分阶段计划**（控制臃肿 —— OpenClaw 失败模式）：
  1. **P1 —— 核心形式模型 + ADR-0001：** 定义 CantiluneGraph + 四个投影 + 函子一致性。无运行时。
  2. **P2 —— 最小执行器 + DAG/Petri 投影 + 静态检查。** 部分评测 C1、C2。
  3. **P3 —— π 演算通信侧面 + A2A 传输。** 评测通信。
  4. **P4 —— 可观测性（trace/重放）+ 评测平台完成。** 评测 C1–C4。
  5. **P5 —— 态射复合侧面 + 重构工具。**
- **回滚：** 每阶段可独立放弃；某阶段评测主张失败即触发重定范围或终止（依据 idea-triage）。

## 14. 兼容性 / 迁移

- 绿地项目；无遗留迁移。互操作目标（A2A、MCP）是兼容性界面，非迁移。
- LangGraph/AutoGen 互操作在 **P1–P2 为非目标**；若评测需要则重新评估。
