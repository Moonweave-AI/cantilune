## 背景

RFC-0001 将 `cantilune` 确立为一个 agent 编排框架，其核心是一个**统一的形式结构**，将四种形式体系作为侧面纳入而非在其中择一。该结构决策是第一个不可逆的架构选择：每一个下游阶段（执行器、通信、可观测性、评测）都依赖于此。

triage 决策（2026-07-23）明确选择了这种统一而非单一形式体系方案，理由是每种形式体系各占编排的一个侧面，且每个侧面映射到一个不同的基线失败模式：

| 形式体系 | 侧面 | 所应对的基线失败模式 |
|---|---|---|
| DAG | 呈现 / 数据流清晰性 | Cursor（固化形态 $\Rightarrow$ 表达力有限） |
| Petri 网 | 并发 / 资源本质 | Codex（缺少显式终止与资源上界证据 $\Rightarrow$ 不可控） |
| π 演算 | 通信本质 | Codex/A2A（非形式化通信 $\Rightarrow$ 不可预知流程） |
| 态射（范畴论） | 复合 / 映射本质 | OpenClaw（无精简核心 $\Rightarrow$ 臃肿） |

## 决策

`cantilune` 采用一个**统一形式对象**，工作名 **`CantiluneGraph`**，并把以四种形式体系为基础的四个投影作为规范性设计目标。这些投影**必须经保持事件身份的函子映射成为一个对象的一致视图**；这是获批条件，不是当前证明状态。

### 投影

1. **DAG 投影** —— 类型化数据依赖图。边承载**数据契约**（schema + 前/后置条件）。拥有表达力与"什么依赖于什么"的呈现。
2. **Petri 投影** —— 并发/控制层。库所建模**资源**（上下文窗口、工具速率限制、人类注意力槽）；变迁在 token 就绪时触发。承载资源/并发不变量及显式终止/活性证明义务；有界性本身不等于终止。
3. **π 演算投影** —— 通信层。agent 间通道作为命名进程；具有动态拓扑与通道移动性的形式语义。拥有 A2A 通信本质。
4. **态射投影** —— 复合层。agent/操作作为范畴中的态射；编排 $=$ 复合。拥有极简精确的复合/重构语义。

### 一致性要求（规范性）

框架**必须**定义投影之间的函子映射，使得一个投影中的事实在其他投影中具有良定义的解释 —— 例如一条 DAG 边对应一条 Petri token 流路径，对应一条 π 演算通道消息，对应一次态射复合。**若无此要求，四个投影就是四个互不相连的模型，"统一结构"的主张即为假。** 该一致性证明是 P1 交付物，也是 ADR-0001 获批的关卡。

**状态限定：** 此段是规范性目标，不是已经成立的定理。下方 2026-07-23 审计附录记录：当前只有态射同一性情形完整，其余规则映射仍开放。

## 考虑过的备选方案

| 备选 | 拒绝理由 |
|---|---|
| 仅 DAG | 无形式化并发/终止（弱于 Codex）；无通信本质；复现"仅 DAG"（等价于 LangGraph） |
| 单一形式体系（Petri / π / 态射） | 失去其他侧面；每个侧面映射到 DRI 希望应对的不同失败模式 |
| 范畴论作为唯一基础 | 最大化通用但抽象、难以关联具体评测成果；对目标生态不友好 |
| 四个互不相连模型由代码粘合 | 复现 OpenClaw 臃肿；无形式一致性；可观测性即结构失败 |

## 后果

**正面（以通过验收关卡为条件）：**
- 只有在相应投影定理实际证明后，才会应对全部四种基线失败模式。
- 相对 LangGraph 可辩护（组合楔子：类型化边 + 资源语义 + 模型解耦路由 + 可观测性即结构）。
- 只有在可观察商 LTS、事件提升/穷尽性关系、共享事件身份、粒度策略与终态谓词均独立定义并证明一致后，可观测性、重放与死锁分类才从形式模型推出。
- 分阶段扩展自然（增量加入投影）。

**负面 / 风险：**
- 规格复杂度：函子一致性要求非平凡，是主要技术风险。**缓解：** 将一致性证明作为 P1 关卡；若无法对全部四者证明，则缩减至可证明一致的子集并记录该缩减。
- 投影无节制累积导致范围/臃肿风险。**缓解：** 分阶段计划（RFC §13）；每个投影必须通过其评测主张。
- 评审人缺口：尚无第二评审人。**缓解：** 将 ADR-0001 获批关卡设为评审人签署。

## ADR-0001 的开放问题（获批前解决）

1. `CantiluneGraph` 的具体语法/类型系统。 —— **部分解决：** 见 `docs/spec/formal-semantics.md` v0.1（SMC $C$ $+$ 重写 $R$）；Q4（策略 DSL）仍开放。
2. 投影之间的函子映射 —— 精确陈述，而非含糊带过。 —— **仍开放，已在 RFC-0002 中拆分。** 原预期为“三者按构造”；审计只确立态射同一性情形。DAG/Petri 重写保持未证；π 的双路线架构已选定，但证明仍开放。
3. 若完整四方一致性无法证明，最小一致子集是什么？（退路） —— **已作为政策解决：** RFC-0002 §6 绑定回退（将 π 缩减至已证子语言，其余标记未经验证）；P1c（完整 π 移动性）明确延后。
4. 策略 DSL 的表达力 vs 终止保证（与 Petri 投影交互）。 —— **仍开放。**
5. 状态同余、行政步策略、可观察推导域与成功终态谓词由何处负责。—— **仍开放；RFC-0002 条款 (4) 所必需。**

## 2026-07-23 后续决策（作为结构性 ADR 的正式记录记于此）

- **统一对象具体化：** $\text{CantiluneGraph} = (C, R)$ —— SMC $C$（静态）$+$ 弦图重写 $R$（动力学）。取代此前的 6-元组草案 $(N, E, S, T, C, M)$，后者是容器而非统一。
- **π 投影范围（half-π (II)）：** request/accept 动态通道创建 + 有限 epoch 的握手后对话与移动性，不含内部递归或 replication。已核实的 FMS 来源使用协变函子范畴，不是此处早前写的逆变预层记号。提议实施已选择“类型化开进程 SMC + 逐点笛卡尔 FMS 指称路线”，并强制要求交换定理；一致性仍**待证，非按构造**。
- **动力学：** 弦图重写（非裸 LTS）。只有在具体事件记录、独立指定的可观察商 LTS、事件提升、终态谓词与所有投影映射均定义并证明后，一个执行事件才预期具有四种形式读法且终态不漂移；当前尚未确立。
- **证明策略：** 明示分期 —— P1a 三个非 π 投影 + P1b π 子语言 + P1c π 完整延后，绑定回退。逐投影状态以 RFC-0002 §3.1 为准，不再沿用原“三者按构造”预期。

本附录只修正证明状态与来源用法；**不**选择替代 π 目标，也不改变统一结构决策。

| 字段 | 结果 |
|---|---|
| 分类 | 形式架构审计；S2；QA-L4；Pre-FCP/M1 |

## 2026-07-23 实施决策附录

在独立审计之后，本轮实施不再把两种目标视为二选一，而是同时选择：

1. 类型化开进程 SMC：接口为对象，复合为 plugging + hiding，张量为并行；
2. 协变 FMS 函子范畴路线：采用逐点笛卡尔张量，π 并行仍是 agent 对象上的内部运算。

获批前必须证明从类型化路线，经擦除后的原生 π，到锁定 FMS 接口的交换定理与观测相容定理。源事件必须对应原生目标单步，不得静默弱化为 $\tau^*$。

本附录只选择证明架构。当前未经复核的 Lean 树已通过 kernel 构建：所呈现
的开进程 SMC、原生擦除、在其映射状态上反射全部原生动作的有限封闭
request/accept 与一次 delegation 证书、有限控制 alpha/结构 late-π、
真正逐点有限幂集 monad、自然 supported-process 支撑语义、有限
`P_f(H-)` 阶段，以及条件式支撑层 FMS 交换定理。支撑 allocation 后接
hiding 现已满足连续自然变换层的回缩等式，但这只是支撑对象定律，不是
FMS agent restriction/coherence package。强化并锁定版本的
`CompleteExternalFMSTheoremPackage` 已写出完整验收接口，但
`CompleteFMSAvailable` 没有 inhabitant。有限参考 π 列的 15 行现均为
原生推导，三项关键 admitted 操作已有精确 event/epoch/replay 轨迹定理，
包括事件真实存储端点、任意有限子段 replay、固定签名 epoch 对齐与从
kernel 推出的有限高度期望界。但当前开放
reconnect/delete 编码还有额外标准 late 环境转移，因此其按事件索引的
证书不是整个 raw late LTS 的完整 reflection。产品范围的一般规则 family
仍未构造。另一个封闭重设计已为四类内部事件给出真正的原生 $\tau$ 步，
并精确分类了这四个源的全部 native 转移；但封闭 open/close 终点仍有
后续 payload $\tau$ 步，Lean 已证明当前每事件两状态的源 LTS 仍不能
完整 reflection。它尚未提供共享四投影
执行包或总
定理。精确范围记录于 `formal/proof-obligations.json`；因此 ADR 仍为
**Proposed**。

**2026-07-23 提议修订：** 请求者已明确授权继续完成整个理论层，因此
有限控制 π 参考演算加入标准、带证明的 mismatch 形式 `[a≠b]P`；
instance reconnect 解释为普通通道 delegation，quiescent instance
delete 解释为 continuation 均为 `0` 的原生 shutdown handshake。三者
均为真正的 π 原生单步，不引入元数据 transition 或弱闭包。此前“缺少
原生见证”的阻断因此被解除：P1c 的 15 个 π 格现均有原生见证；但更强的
完整 late-reflection 阻断并未解除，因为开放 reconnect/delete handshake
仍有环境转移。有限参考演算随后闭合了全部 60 个按事件索引的格与四份
受限操作证书。ADR 在 Owner/DRI 与进程
语义评审完成前仍为 Proposed；一般 admission 规则及静态/资源/admission
总定理仍开放。

获批前所需证据：

1. 定义精确源语法、配置、规则、新鲜性与粒度；
2. 为现已明确的完整 FMS powerdomain/domain/action/hiding/全抽象接口
   构造并独立核验真实 inhabitant；
3. 将已实现的 SMC/参考证书扩展到完整获准规则集；
4. 独立定义可观察目标推导、状态同余、行政步策略与事件提升/穷尽性关系；
5. 定义并证明成功终态谓词的保持与反射；
6. 完成 DAG/Petri 直接规则映射证明；以及
7. 取得独立形式数学/范畴/进程语义评审。

## 可验证性

本 ADR 为**提议**，非已批准。其主张（尤其是函子一致性）在 P1 交付物中演示之前均属**未经验证**。依据治理，在 ADR-0001 获批之前，除 P1 范围界定工作外不应推进任何实现。
