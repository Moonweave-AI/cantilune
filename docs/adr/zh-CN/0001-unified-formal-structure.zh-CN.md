# ADR-0001：Cantilune 的统一形式结构

| 字段 | 值 |
|---|---|
| 状态 | **提议**（等待 RFC-0001 的第二评审人） |
| 日期 | 2026-07-23 |
| 决策负责人 | Joker-of-Gotham（DRI） |
| 评审人 | 待定（治理缺口 —— 见 RFC-0001 §0） |
| 相关 | RFC-0001 |
| 风险 | S2 |

## 背景

RFC-0001 将 `cantilune` 确立为一个 agent 编排框架，其核心是一个**统一的形式结构**，将四种形式体系作为侧面纳入而非在其中择一。该结构决策是第一个不可逆的架构选择：每一个下游阶段（执行器、通信、可观测性、评测）都依赖于此。

triage 决策（2026-07-23）明确选择了这种统一而非单一形式体系方案，理由是每种形式体系各占编排的一个侧面，且每个侧面映射到一个不同的基线失败模式：

| 形式体系 | 侧面 | 所应对的基线失败模式 |
|---|---|---|
| DAG | 呈现 / 数据流清晰性 | Cursor（固化形态 ⇒ 表达力有限） |
| Petri 网 | 并发 / 资源本质 | Codex（无终止/有界性 ⇒ 不可控） |
| π 演算 | 通信本质 | Codex/A2A（非形式化通信 ⇒ 不可预知流程） |
| 态射（范畴论） | 复合 / 映射本质 | OpenClaw（无精简核心 ⇒ 臃肿） |

## 决策

`cantilune` 采用一个**统一形式对象**，工作名 **`CantiluneGraph`**，它允许四个良定义的投影，每个投影以一种形式体系为奠基。四个投影通过函子映射成为**一个对象的一致视图**，而非四个互不相连的模型。

### 投影

1. **DAG 投影** —— 类型化数据依赖图。边承载**数据契约**（schema + 前/后置条件）。拥有表达力与"什么依赖于什么"的呈现。
2. **Petri 投影** —— 并发/控制层。库所建模**资源**（上下文窗口、工具速率限制、人类注意力槽）；变迁在 token 就绪时触发。拥有有界并发、资源边界、终止/活性。
3. **π 演算投影** —— 通信层。agent 间通道作为命名进程；具有动态拓扑与通道移动性的形式语义。拥有 A2A 通信本质。
4. **态射投影** —— 复合层。agent/操作作为范畴中的态射；编排 = 复合。拥有极简精确的复合/重构语义。

### 一致性要求（规范性）

框架**必须**定义投影之间的函子映射，使得一个投影中的事实在其他投影中具有良定义的解释 —— 例如一条 DAG 边对应一条 Petri token 流路径，对应一条 π 演算通道消息，对应一次态射复合。**若无此要求，四个投影就是四个互不相连的模型，"统一结构"的主张即为假。** 该一致性证明是 P1 交付物，也是 ADR-0001 获批的关卡。

## 考虑过的备选方案

| 备选 | 拒绝理由 |
|---|---|
| 仅 DAG | 无形式化并发/终止（弱于 Codex）；无通信本质；复现"仅 DAG"（等价于 LangGraph） |
| 单一形式体系（Petri / π / 态射） | 失去其他侧面；每个侧面映射到 DRI 希望应对的不同失败模式 |
| 范畴论作为唯一基础 | 最大化通用但抽象、难以关联具体评测成果；对目标生态不友好 |
| 四个互不相连模型由代码粘合 | 复现 OpenClaw 臃肿；无形式一致性；可观测性即结构失败 |

## 后果

**正面：**
- 按构造应对全部四种基线失败模式。
- 相对 LangGraph 可辩护（组合楔子：类型化边 + 资源语义 + 模型解耦路由 + 可观测性即结构）。
- 可观测性与重放从形式模型自然得出。
- 分阶段扩展自然（增量加入投影）。

**负面 / 风险：**
- 规格复杂度：函子一致性要求非平凡，是主要技术风险。**缓解：** 将一致性证明作为 P1 关卡；若无法对全部四者证明，则缩减至可证明一致的子集并记录该缩减。
- 投影无节制累积导致范围/臃肿风险。**缓解：** 分阶段计划（RFC §13）；每个投影必须通过其评测主张。
- 评审人缺口：尚无第二评审人。**缓解：** 将 ADR-0001 获批关卡设为评审人签署。

## ADR-0001 的开放问题（获批前解决）

1. `CantiluneGraph` 的具体语法/类型系统。
2. 投影之间的函子映射 —— 精确陈述，而非含糊带过。
3. 若完整四方一致性无法证明，最小一致子集是什么？（退路）
4. 策略 DSL 的表达力 vs 终止保证（与 Petri 投影交互）。

## 可验证性

本 ADR 为**提议**，非已批准。其主张（尤其是函子一致性）在 P1 交付物中演示之前均属**未经验证**。依据治理，在 ADR-0001 获批之前，除 P1 范围界定工作外不应推进任何实现。
